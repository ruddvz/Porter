Porter

GitHub Pages only serves this static page. The seller dashboard, admin console, APIs, and WhatsApp webhooks live in the Next.js app, which must be deployed separately (this repo is set up for Vercel).

Run the real Porter app

One click imports this repo into Vercel. After the first deploy finishes, open your *.vercel.app URL, copy every variable from .env.example into the Vercel project Settings → Environment Variables, and set NEXT_PUBLIC_APP_URL to that same URL so auth redirects and /track/… links work.

Deploy on Vercel Env checklist (README)

Where to see the real UI

  • Seller dashboard — Kanban, orders, settings, analytics (after login).
  • Public order tracking — /track/<slug> (link sent in WhatsApp after checkout).
  • Admin console — Platform overview, live activity feed, CSV export link.

Optional: in this GitHub repo, add a variable PORTER_LIVE_URL (Settings → Secrets and variables → Actions → Variables) with your production URL; the next Pages deploy will show an Open live app button above.
